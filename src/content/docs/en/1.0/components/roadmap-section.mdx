The Roadmap Section displays roadmaps, timelines, or milestones in chronological order. Perfect for showing project plans, development timelines, or important milestones in a clear, visual format.

<Aside>
  **Best for:** Project roadmaps, timelines, milestones, development plans,
  future plans, chronological event displays.
</Aside>

## Fields

### Headline

* **Type:** String (required)
* **Max Length:** 100 characters
* **Purpose:** Main heading for the roadmap section
* **Example:** "Our Roadmap" or "Development Timeline"

### Description

* **Type:** Text, 3 rows (optional)
* **Max Length:** 500 characters
* **Purpose:** Description for the roadmap section
* **Example:** "Our planned milestones and developments"

### Timeline Items

* **Type:** Array of timeline item objects (optional)
* **Min Count:** 1
* **Max Count:** 20
* **Purpose:** Timeline items for the roadmap

#### Timeline Item Fields

**Title** (required)

* Title of the milestone or feature
* Max Length: 100 characters
* Example: "Q1 2025: New Vehicles"

**Description** (optional)

* Detailed description of the milestone
* Type: Text, 3 rows
* Max Length: 500 characters
* Example: "Integration of electric vehicles into our fleet"

### Background Color

* **Type:** Reference to `backgroundColor` (optional)
* **Purpose:** Sets the background color for the entire roadmap section

***

## Usage Example

Here's a typical Roadmap Section:

**Headline:** "Our Roadmap"

**Description:** "Our planned milestones and developments"

**Timeline Items:**

1. **"Q1 2025: New Vehicles"**

   * Description: "Integration of electric vehicles into our fleet"

2. **"Q2 2025: Online Registration"**

   * Description: "Introduction of an online registration function"

3. **"Q3 2025: Expanded Locations"**
   * Description: "Opening of new locations in the region"

***

## Best Practices

**Chronology:** Arrange items in chronological order for clarity

**Clarity:** Use clear, concise titles that communicate the milestone

**Descriptions:** Provide detailed but concise descriptions

**Count:** 3-10 items are optimal for readability

**Regular updates:** Keep the roadmap current with actual progress

**Consistency:** Use consistent date formats and terminology

***

## Key Features

**Chronological display:** Items displayed in timeline order

**Clear structure:** Title and description for each milestone

**Visual timeline:** Timeline visualization for easy understanding

**Flexible content:** Support for multiple timeline items

**Responsive:** Adapts to different screen sizes

***

## Where It's Used

The Roadmap Section appears in these content types:

* **Pages** (`page`) – General content pages, especially project or development pages

***

## Technical Notes

* Timeline items are displayed in the order they appear in the array
* Items are typically displayed in a vertical timeline format
* Background color references the global `backgroundColor` schema
* The section uses semantic HTML for accessibility
* Timeline visualization is handled by the frontend application
