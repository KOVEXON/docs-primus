The Event Display Section presents events from an events list in an organized table format. It's perfect for displaying courses, appointments, or any scheduled activities with clear date, time, and availability information.

<Aside>
  **Best for:** Event calendars, course overviews, appointment schedules, class
  listings, training schedules.
</Aside>

## Fields

### Section Headline

* **Type:** String (optional)
* **Purpose:** Main heading for the events section
* **Example:** "Upcoming Events" or "Course Schedule"

### Section Description

* **Type:** Text, 3 rows (optional)
* **Purpose:** Optional description below the headline
* **Example:** "Here you'll find all current courses and appointments"

### Date Column Title

* **Type:** String (optional)
* **Default Value:** "Day & Date"
* **Purpose:** Title for the date column
* **Example:** "Date" or "Day & Date"

### Title Column Title

* **Type:** String (optional)
* **Default Value:** "Session"
* **Purpose:** Title for the session title column
* **Example:** "Course" or "Event"

### Time Column Title

* **Type:** String (optional)
* **Default Value:** "Time"
* **Purpose:** Title for the time column
* **Example:** "Time" or "Start Time"

### Available Status Text

* **Type:** String (optional)
* **Default Value:** "Spots available"
* **Purpose:** Text displayed when spots are available
* **Example:** "Spots available" or "Open"

### Full Status Text

* **Type:** String (optional)
* **Default Value:** "Fully booked"
* **Purpose:** Text displayed when the event is fully booked
* **Example:** "Fully booked" or "Sold out"

### Events List

* **Type:** Reference to `eventsList` (required)
* **Purpose:** Reference to an events list document

### No Events Message

* **Type:** String (optional)
* **Purpose:** Message displayed when no events are available
* **Example:** "Currently no events are scheduled"

### Background Color

* **Type:** Reference to `backgroundColor` (optional)
* **Purpose:** Sets the background color for the entire events section

## Best Practices

**Clarity:** Use clear, understandable column titles that match your content

**Status texts:** Use clear status texts that users can quickly understand

**Regular updates:** Keep the events list current with upcoming events

**Empty state:** Provide a friendly message when no events are available

**Consistency:** Use consistent terminology across all column titles and status texts

**Accessibility:** Ensure the table structure is accessible with proper headers

***

## Key Features

**Table format:** Events displayed in a clear, scannable table layout

**Status display:** Shows availability status for each event

**Centralized management:** Uses events list documents for centralized event management

**Customizable columns:** Column titles can be customized to match your needs

**Empty state handling:** Friendly message when no events are available

***

## Where It's Used

The Event Display Section appears in these content types:

* **Pages** (`page`) – Event pages, course pages, and general content pages

***

## Technical Notes

* Events are fetched from the referenced events list document
* Table structure uses semantic HTML for accessibility
* Status is automatically calculated based on event capacity and registrations
* Background color references the global `backgroundColor` schema
* Events are typically sorted by date (oldest to newest or vice versa)
