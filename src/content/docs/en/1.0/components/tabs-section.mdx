## Overview

**Tabs Section** organizes content into different tabs using tab navigation. It's perfect for structured, categorized information that benefits from a clear, organized presentation where only one section is visible at a time.

<Aside>
  **Best for:** Categorized content, product details, service pages, price
  comparisons, information pages, feature presentations.
</Aside>

***

## Fields

### Headline

**Type:** String\
**Required:** Yes

Main heading for the entire tabs section.

**Examples:** `Our Service Packages`, `Service Options`

***

### Description

**Type:** Text\
**Required:** No\
**Max Length:** 500 characters

Description for the tabs section.

**Example:** `Choose the right service package for your needs`

***

### Items

**Type:** Array of objects\
**Required:** Yes (minimum 1)

List of tab items displayed as tabs.

Each tab item includes:

#### Item Headline

* **Type:** String
* **Required:** Yes
* **Description:** Headline of the tab (displayed as tab label)
* **Example:** `Basic Package`

#### Icon (mobile version only)

* **Type:** Lucide Icon
* **Required:** No
* **Description:** Optional icon (mobile version only)
* **Source:** Lucide icon library
* **Examples:** Package, Star, Crown, Building

#### Content

* **Type:** Rich Text Section (`richTextSection`)
* **Required:** Yes
* **Description:** Rich text content displayed in the active tab

***

### Background Color

**Type:** Reference to `backgroundColor`\
**Required:** No

Sets the background color for the entire tabs section.

***

## Usage Example

### Headline

`Our Service Packages`

### Description

`Choose the right service package for your needs`

### Items (Tabs)

1. **"Basic Package"** (Package icon)

   * Content: "The Basic Package includes essential services for your needs..."

2. **"Standard Package"** (Star icon)

   * Content: "With the Standard Package, you get comprehensive services with additional features..."

3. **"Premium Package"** (Crown icon)

   * Content: "The Premium Package is designed for businesses requiring advanced solutions..."

4. **"Enterprise Package"** (Building icon)
   * Content: "With the Enterprise Package, you get full-scale solutions with dedicated support..."

## Best Practices

* **Headline:** Use concise, meaningful headlines for each tab
* **Categorization:** Group related content logically in tabs
* **Icons:** Use icons consistently and only when they add value (mobile version only)
* **Content length:** Keep content per tab focused and balanced
* **Count:** 3-5 tabs are optimal for user experience
* **Equivalence:** Tabs should contain equivalent content
* **Background color:** Use background colors sparingly to ensure readability

***

## Key Features

* Tab navigation for organized content display
* Rich text content with full formatting capabilities
* Optional icons for visual support (mobile version only)
* Structured display ideal for categorized, equivalent content
* Space-efficient layout through tab navigation
* Interactive tab switching for users

## Technical Notes

* Only one tab is active at a time
* Content uses the `richTextSection` type for full formatting capabilities
* Icons (mobile version only) come from the Lucide icon library
* Background color references the global `backgroundColor` schema
* The section supports keyboard navigation for accessibility

***

## Differences from Other Content Blocks

| Feature         | Tabs Section     | Accordion Section   |
| --------------- | ---------------- | ------------------- |
| Structure       | Fixed (Tabs)     | Fixed (Accordion)   |
| Navigation      | Tab navigation   | Expandable          |
| Visibility      | One tab active   | Multiple expandable |
| Rich Text       | Yes (per tab)    | Yes (per item)      |
| Icons           | No               | Yes (optional)      |
| Headlines       | Yes (per tab)    | Yes (per item)      |
| Space-efficient | Yes              | Yes                 |
| Usage           | Categorized info | Structured info     |

***

## When to Use Tabs vs. Accordion

### Use Tabs Section when:

* Content is equivalent and categorized
* Only one area should be visible at a time
* Content belongs together thematically
* Clear navigation is important

### Use Accordion Section when:

* Content is hierarchically structured
* Multiple areas can be visible simultaneously
* Content varies in length
* Expandable areas are preferred
