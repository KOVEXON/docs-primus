Die **Feature Section** ist ein Content-Block zur Präsentation von Features, Vorteilen oder Dienstleistungen mit Icons, Headlines und Beschreibungen.

## Felder

### Headline

- **Typ**: String
- **Pflichtfeld**: Nein
- **Beschreibung**: Hauptüberschrift für die Feature-Sektion

### Description

- **Typ**: Text
- **Pflichtfeld**: Nein
- **Beschreibung**: Beschreibung für die Feature-Sektion

### Features

- **Typ**: Array von Feature-Objekten
- **Pflichtfeld**: Nein
- **Beschreibung**: Liste der Features

#### Feature-Objekt Felder

##### Headline

- **Typ**: String
- **Pflichtfeld**: Nein
- **Beschreibung**: Überschrift des Features

##### Subheadline

- **Typ**: String
- **Pflichtfeld**: Nein
- **Beschreibung**: Unterüberschrift des Features

##### Description

- **Typ**: Text
- **Pflichtfeld**: Nein
- **Beschreibung**: Beschreibung des Features

##### Icon

- **Typ**: Lucide Icon
- **Pflichtfeld**: Nein
- **Beschreibung**: Optionales Icon für das Feature

##### Call to Action Buttons

- **Typ**: Reference zu `button`
- **Pflichtfeld**: Nein
- **Beschreibung**: Button-Konfiguration für das Feature

### Background Color

- **Typ**: Reference zu `backgroundColor`
- **Pflichtfeld**: Nein
- **Beschreibung**: Hintergrundfarbe für die gesamte Feature-Sektion

## Verwendung

Die Feature Section wird typischerweise verwendet für:

- **Feature-Präsentationen** - Hauptfeatures von Produkten oder Dienstleistungen
- **Vorteile** - Vorteile eines Angebots
- **Dienstleistungen** - Übersicht von Dienstleistungen
- **Angebote** - Präsentation verschiedener Angebote

## Beispiel

Eine typische Feature Section könnte folgende Struktur haben:

### Headline

"Unsere Vorteile"

### Description

"Warum Sie sich für uns entscheiden sollten"

### Features

**Feature 1:**

- **Headline**: "Erfahrene Fahrlehrer"
- **Subheadline**: "Über 10 Jahre Erfahrung"
- **Description**: "Unsere Fahrlehrer verfügen über langjährige Erfahrung..."
- **Icon**: UserCheck
- **CTA Button**: "Mehr erfahren" → `/ueber-uns`

**Feature 2:**

- **Headline**: "Moderne Fahrzeuge"
- **Subheadline**: "Elektro und Hybrid"
- **Description**: "Lernen Sie mit modernsten Fahrzeugen..."
- **Icon**: Car
- **CTA Button**: "Fahrzeuge ansehen" → `/fahrzeuge`

**Feature 3:**

- **Headline**: "Flexible Termine"
- **Subheadline**: "Auch am Wochenende"
- **Description**: "Wir bieten flexible Termine an..."
- **Icon**: Calendar
- **CTA Button**: "Termin buchen" → `/termin-buchen`

## Best Practices

- **Konsistenz**: Verwenden Sie ähnliche Strukturen für alle Features
- **Icons**: Verwenden Sie Icons konsistent und passend zum Feature
- **Klarheit**: Kurze, prägnante Headlines und Subheadlines
- **Anzahl**: 3-6 Features sind optimal für die Benutzererfahrung
- **Buttons**: Verwenden Sie CTA-Buttons sparsam, nur wenn relevant
